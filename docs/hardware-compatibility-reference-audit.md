# Hardware Compatibility Reference Audit

Date: 2026-07-08

This audit compares the current qurbrix-hwinfo implementation with:

- `../ReferenceProject/deepin-devicemanager-6.0.67`
- `../ReferenceProject/kylin-os-manager-build-2.0.0-76update2`

Scope: verify each component parser/probe against the two references, with special attention to fallback and exception-handling practices.

## Summary

Current qurbrix-hwinfo has absorbed several P0/P1 compatibility gaps that were previously missing:

- CPU uses `lscpu`, `lshw -class processor`, and `dmidecode -t 4` as optional sources, accepts both util-linux `lscpu` title-style keys and Deepin's proc-style lowercase CPU map keys for identity/family/model/stepping/bogomips/flags, treats Deepin-style source placeholder values such as `N/A`, `Null`, `none`, `Not Provided`, `Not Specified`, `Default string`, `Unspecified`, and `Unknown` as empty, emits `source_empty` when those command sources succeed but contain no usable CPU fields, merges DMI counts/speeds/family plus Kylin-visible slot, serial, external-clock, and `Core Enabled` fields, preserves DMI `Family` as a supplementary field when another CPU source proves the device even if that DMI record is not strong enough to count as device evidence, protects Loongson names, strips Deepin-style trailing `x <count>` CPU instance suffixes from merged CPU names, exposes `Thread(s) per core`, lscpu `On-line CPU(s) list` as online logical thread count, Kylin-style online core count, and DMI enabled core count, infers missing or contradictory `Core(s) per socket` from `CPU(s)`/`Thread(s) per core`/`Socket(s)`, and normalizes CPU vendors/architectures, including Kylin-style D2000-only Phytium model-name inference plus common domestic vendor/manufacturer strings such as Loongson, Phytium, Hygon, Zhaoxin, Huawei/HiSilicon, and Sunway.
- CPU also reads `/proc/cpuinfo` as an optional procfs fallback, using Kylin-style `Hardware`/`Processor` fields, LoongArch-style `cpu model`, x86 `cpu cores`/`siblings`/`physical id` topology fields, `cpu MHz`/`CPU MHz` plus Deepin-style lowercase `cpu mhz` current-frequency fields, `Features` plus lowercase `features`, `cache size` as an L2 cache fallback, `clflush size` as a structured cache-line-size field, and Deepin-visible ARM CPU identity fields (`CPU implementer`, `CPU architecture`, `CPU variant`, `CPU part`, `CPU revision`) when command sources are missing or incomplete; ARM-style generic `Processor: AArch64 Processor rev ...` no longer prevents a later `Hardware` model from becoming the CPU name, and readable-but-empty proc cpuinfo parses now emit `source_empty`.
- CPU reads `/proc/hardware` as an optional procfs fallback for Kylin's Kirin 990/9006C detection path, emitting `source_empty` when a readable file has no recognized hardware fields.
- CPU uses Kylin-style `/sys/devices/system/cpu/*/cpufreq` frequency files as a low-priority fallback for max/min/current frequency when `lscpu` and DMI do not provide those fields, including `scaling_max_freq`/`scaling_min_freq` as fallbacks when `cpuinfo_max_freq`/`cpuinfo_min_freq` are unavailable and averaging all readable per-core `scaling_cur_freq` values before falling back to `cpu0`.
- CPU also reads Deepin-style `/sys/devices/system/cpu/cpu*/topology`, `/sys/devices/system/cpu/online`, and `cpu0/cache/index*` as a low-priority sysfs fallback, preserving aggregate threads/sockets/cores/threads-per-core, online threads/cores, and L1d/L1i/L2/L3/L4 cache totals when command/procfs CPU sources are missing or incomplete.
- CPU preserves Deepin/Kylin-visible `lscpu` cache fields (`L1d cache`, `L1i cache`, `L2 cache`, `L3 cache`, `L4 cache`), keeping newer total-with-instances strings as-is and totaling older per-shared-group cache strings from `L*d shared cpu list` fields, and derives Deepin-style CPU extension names from `flags`.
- CPU intentionally keeps a generic Linux source-priority strategy instead of branching on Deepin/Kylin OS IDs: per-logical-CPU IDs remain out of the static inventory model, while aggregate L4 cache is exposed when `lscpu` or sysfs provides it; Kylin's DMI `Core Enabled` and lscpu `On-line CPU(s) list`/`Online Cores` fields are inventory-like and are exposed as `CpuInfo.enabled_cores`, `CpuInfo.online_threads`, and `CpuInfo.online_cores`, while Kylin's governor, available-frequency, sensor, and temperature/fan paths are runtime monitoring/control features rather than static CPU inventory fields.
- Monitor uses `xrandr --query`, `xrandr --verbose`, `/sys/class/drm/*/edid`, and optional `hwinfo --monitor`, parses EDID in-process, preserves identity/date/size/preferred mode/gamma/diagonal inches and xrandr max mode, uses a unique readable, unique `status=connected`, unique `enabled=enabled`, or unique EDID preferred-mode match for the current xrandr resolution when duplicate normalized connectors exist, and keeps or creates monitor devices when EDID parsing fails or xrandr query output parses empty.
- GPU uses `lspci -nn -k`, normalizes common and domestic GPU vendor names, falls back to PCI vendor IDs when lspci text is generic, optionally enriches matched PCI adapters from `lshw -class display`, enriches matched DRM adapters with VRAM totals from `/sys/class/drm/*/device/mem_info_vram_total`, optionally enriches PCI-addressed VRAM totals from `dmesg`, optionally enriches Deepin/Jingjia VRAM totals from PCI `gpu-info` or unique Jingjia `/proc/gpuinfo_0`, optionally enriches NVIDIA VRAM totals from `nvidia-smi` by PCI bus ID or `nvidia-settings -q VideoRam` on unique-NVIDIA-GPU systems, optionally enriches a sole GPU, unique vendor-matched GPU, or unique renderer/model-matched same-vendor GPU from Deepin-style `glxinfo -B` OpenGL renderer/vendor/version data, and consumes display-class `/sys/bus/pci/devices/*` nodes with kernel driver data when `lspci` cannot run or parses no display controllers.
- Source execution has structured `Missing`, `PermissionDenied`, `Timeout`, and `Failed` classifications, which is cleaner than most reference scripts and should remain the project-wide pattern.
- Command execution forces `LC_ALL=C`, `LANG=C`, and `LANGUAGE=en`, absorbing Deepin/Kylin's locale-stabilization practice for English-key parsers.
- System probing records runtime hostname/OS/kernel/architecture plus DMI `System Information` fields from `dmidecode -t 1`, and falls back to `/sys/class/dmi/id` product fields when dmidecode is unavailable.
- BIOS/motherboard probing falls back to `/sys/class/dmi/id` when `dmidecode` is missing or denied, preserves baseboard version/asset/location, chassis identity/state fields, Physical Memory Array fields, and BIOS Language Information where available, and BIOS probing now records Linux firmware type plus Secure Boot state from `/sys/firmware/efi` when available.
- Memory probing enriches successful `dmidecode -t memory` DIMM records from matching `lshw -class memory` banks by locator or serial, preserving lshw product/description, vendor, size, clock, width, and source evidence while retaining DMI-only details; it also falls back to lshw DIMM banks when DMI cannot run. It uses Deepin-style part/product as the memory device name when available and lshw description when product is missing, preserves DMI `Bank Locator` as both a slot fallback and a structured field, preserves DMI `Manufacturer ID` / `Module Manufacturer ID` as a raw vendor fallback when `Manufacturer` is empty or unspecified, preserves `Configured Memory Speed` independently and uses it when `Speed` is missing or unknown, preserves DMI total/data width, min/max/configured voltage, Deepin-visible DIMM detail fields including `Error Information Handle` and `Set`, Module/Subsystem IDs, and NVDIMM size fields, also maps Physical Memory Array data into a memory aggregate device when present, drops Deepin-style `<OUT OF SPEC>` memory types, normalizes MB/MiB/GiB display sizes to GB with Deepin-style SW64 MiB odd-GB compensation, then tries `decode-dimms` SPD data, raw DDR4/DDR5 SPD EEPROM/nvmem sysfs data with a common manufacturer-ID friendly-name map, EDAC sysfs DIMM nodes, Kylin FT1500a `/sys/phytium1500a_info/memory*`, device-tree memory reg, and finally `/proc/meminfo` total memory if DIMM-level sources are unavailable. DDR5 raw SPD size/speed deep decoding is not treated as a local Deepin/Kylin parity target.
- Battery probing falls back to `/sys/class/power_supply/BAT*` when `upower --dump` cannot run, preserves sysfs temperature, and normalizes the common `LGC` battery vendor alias to `LG Chem`.
- Network probing uses `ip -j link` plus optional `ip -j addr`, enriches interfaces from `/sys/class/net/*` with speed, duplex, wireless/ethernet type/capability marking, kernel driver/modules, and PCI bus identity from uevent when available, optionally uses `lshw -class network` for human-readable product/vendor/capacity/driver/firmware data, and falls back to sysfs interfaces when `ip -j link` cannot run.
- Storage probing requests WWN/firmware revision from `lsblk`, enriches kernel driver data from `/sys/block/*/device/uevent`, preserves PCI controller identity from sysfs uevent data including `/sys/class/nvme/<controller>/device/uevent`, canonical `/sys/block/<disk>/device` PCI ancestor paths, bounded SATA/SCSI parent `uevent` traversal, a unique storage-class `/sys/bus/pci/devices/*` fallback for SATA/ATA/SCSI disks, and a unique media-type-matching PCI storage controller fallback when several storage-class controllers exist, optionally enriches human-readable disk identity from `lshw -class disk`, controller vendor/model/driver identity from `lshw -class storage` or `lspci -nn -k`, disk identity/driver data from `hwinfo --disk`, and Kylin-style `hdparm -i`, optionally enriches Deepin-style disk model/serial/firmware identity, SMART health, temperature, and NVMe health counters from `smartctl -a -j`, infers Kylin-style common disk model-prefix vendors such as WDC/Western Digital, ST/Seagate, and FORESEE/Foresee plus Longsys for RS-prefixed models when vendor is missing, and falls back to `/sys/block/*` or hwinfo-only disk records when `lsblk` cannot run or parses no disk records.
- USB probing enriches `lsusb` records from `/sys/bus/usb/devices/*` when bus/device numbers match, including max power, optionally uses `lsusb -v` for the first interface descriptor class/subclass/protocol, and falls back to sysfs devices when `lsusb` cannot run.
- Bluetooth probing falls back to `/sys/class/bluetooth/hci*` when `hciconfig -a` cannot run, preserving controller address and rfkill power state when available, warns when successful `hciconfig` output contains no parsable controllers, and optionally uses `lshw -class communication` for human-readable controller identity/driver data.
- Audio probing preserves PCI bus identity and kernel module names from `/sys/class/sound/card*/device/uevent` and `/driver/module/drivers/*` when available, optionally uses `lshw -class multimedia` and `hwinfo --sound` for human-readable audio product/vendor/driver data, and optionally uses `pactl list cards` for ALSA card profile names.
- Camera probing falls back to `/sys/class/video4linux/video*` when `v4l2-ctl --list-devices` cannot run, enriches cameras with video4linux sysfs driver and USB identity/speed data when available, optionally uses `lshw -class multimedia` records matched by `/dev/video*` logical name for human-readable identity/driver data, and optionally preserves `v4l2-ctl --list-formats-ext` format/resolution capabilities.
- Input probing falls back to `/sys/class/input/event*` when `/proc/bus/input/devices` cannot be read or parses no devices, uses evdev capability bitmasks from procfs/sysfs to classify touchscreens, touchpads, tablets, and mice when names/handlers are not enough, and optionally uses `hwinfo --keyboard`/`--mouse` for human-readable keyboard/mouse identity and driver data.
- CD-ROM probing enriches `/proc/sys/dev/cdrom/info` drives from `/sys/class/block/sr*`, optional `lshw -class disk` `*-cdrom` records, and optional `hwinfo --cdrom` records with basic vendor/model/serial/firmware/driver identity, and falls back to sysfs or hwinfo drives when proc cdrom info cannot be read or parses no drives.
- Printer probing falls back to `lpstat -v` when `lpstat -a` cannot run or parses no queues, uses `lpstat -d` to mark the default destination when available, and uses `lpstat -l -p` to preserve queue make/model descriptions when available.
- PCI probing falls back to `/sys/bus/pci/devices/*`, including available kernel driver data, when `lspci -nn -k` cannot run.

Confirmed defects fixed during this audit:

- `CPU MHz` from `lscpu` is now used as `current_freq_mhz` when DMI current speed is unavailable.
- `Thread(s) per core` from `lscpu` is now exposed through `CpuInfo` and used to infer missing or contradictory `Core(s) per socket` when `CPU(s)` and `Socket(s)` make the core count unambiguous.
- Kylin-style online core count is now exposed through `CpuInfo.online_cores`, derived from `lscpu` online logical CPUs plus `Thread(s) per core` or from sysfs `/sys/devices/system/cpu/online` plus CPU topology.
- CPU family from `lscpu` or DMI, plus model/stepping/bogomips/virtualization, Deepin-style extensions, and L1d/L1i/L2/L3/L4 cache fields parsed or derived from `lscpu`, are now exposed through `CpuInfo`.
- Older Deepin-style `lscpu` cache output such as `L1d cache: 32K` plus `L1d shared cpu list: 0,4` is now totaled before being exposed through `CpuInfo`, including L4 when present.
- Monitor sysfs fallback now works when `xrandr --query` is missing, but does not create fake devices from empty/bad sysfs EDID.
- sysfs EDID read failures now produce source warnings instead of being silently ignored.
- Monitor EDID parsing now preserves gamma and computes diagonal inches from physical size.
- Monitor xrandr query parsing now preserves the first advertised mode as max resolution.
- Monitor probing now uses optional `hwinfo --monitor` to enrich safely matched monitors or create hwinfo-only fallback monitor devices with vendor/model/serial/size data.
- Monitor probing now uses the sole readable sysfs EDID, the sole readable `status=connected` sysfs EDID, the sole readable `enabled=enabled` sysfs EDID, or a sole EDID preferred-mode match for the current xrandr resolution for duplicate normalized connector paths while still avoiding ambiguous matches when these signals cannot uniquely identify a readable candidate.
- GPU vendor normalization now uses numeric PCI vendor IDs when text is only generic `Device`.
- `lshw -class display` enrichment now preserves human-readable GPU product/model, vendor, and driver data by matching PCI bus addresses, including on sysfs GPU fallback devices.
- DRM sysfs enrichment now preserves GPU VRAM totals from `mem_info_vram_total` by matching DRM card PCI slots back to GPU devices.
- `dmesg` enrichment now preserves Deepin-style PCI-addressed GPU VRAM totals such as `0000:03:00.0: VRAM: 8192M` when DRM sysfs VRAM data is unavailable.
- `glxinfo -B` enrichment now preserves Deepin-style OpenGL renderer/vendor/version data on single-GPU systems, multi-GPU systems where the OpenGL vendor matches exactly one GPU, and same-vendor multi-GPU systems where renderer model tokens match exactly one GPU, without assigning the active renderer to still-ambiguous devices.
- Deepin-style PCI `gpu-info` and unique Jingjia `/proc/gpuinfo_0` enrichment now preserve vendor-specific GPU VRAM totals when common DRM/dmesg paths are unavailable.
- NVIDIA enrichment now preserves GPU VRAM totals by matching `nvidia-smi` queried PCI bus IDs to GPU devices, with `nvidia-settings -q VideoRam` as a unique-NVIDIA-GPU fallback.
- Empty successful `lspci -nn -k` GPU parses now emit `source_empty` and still try display-class `/sys/bus/pci/devices/*` fallback devices.
- Loopback/common virtual network interfaces, UPower line-power devices, USB root hubs/hubs, empty BIOS DMI output, and duplicate UVC camera video nodes are now filtered.
- Malformed `ip -j link` / `lsblk -J` output and failed optional Bluetooth/printer enrichment sources now produce warnings instead of silently returning incomplete data.
- Empty successful CPU command parses from `lscpu`, `lshw -class processor`, or `dmidecode -t 4`, plus readable-but-empty `/proc/cpuinfo` or `/proc/hardware`, now emit `source_empty` while still allowing lower-priority CPU sources such as CPU sysfs topology/cache/cpufreq data to produce the CPU device.
- CPU parser now ignores Deepin-style placeholder values (`N/A`, `Null`, `none`, `Not Provided`, `Not Specified`, `Default string`, `Unspecified`, and `Unknown`), matching Deepin's behavior of rejecting invalid display values instead of treating them as authoritative CPU names or vendors.
- `/proc/cpuinfo` fallback now covers ARM-style `Hardware`/`Processor`, Kylin-style `cpu MHz`/`CPU MHz`, Deepin-style lowercase `cpu mhz`, `Features`/`features`, `BogoMIPS`, and logical processor counts.
- `/proc/cpuinfo` fallback now preserves Deepin-visible ARM CPU identity fields (`CPU implementer`, `CPU architecture`, `CPU variant`, `CPU part`, `CPU revision`) as structured `CpuInfo` fields while still normalizing the generic architecture to values such as `aarch64`.
- `/proc/cpuinfo` fallback now uses Kylin-style `cpu cores`/`siblings` plus available `physical id` values to preserve CPU topology when `lscpu` is unavailable.
- `/proc/cpuinfo` fallback now preserves Kylin-style `cache size` as `CpuInfo.l2_cache` when `lscpu` cache data is unavailable.
- `/proc/cpuinfo` fallback now preserves Kylin-style `clflush size` as `CpuInfo.clflush_size_bytes`.
- DMI type 4 parsing now preserves Kylin-visible `Socket Designation`, `Serial Number`, and `External Clock` as CPU structured fields.
- DMI type 4 `Family` now fills `CpuInfo.family` as supplemental data when another CPU source exists, while family-only DMI output still emits `source_empty` and does not count as a successful CPU source by itself.
- CPU vendor inference now maps bare `D2000` model names to Phytium, matching Kylin's Phytium/D2000 rule even when the model string does not contain `Phytium`.
- CPU vendor normalization now also accepts common domestic vendor/manufacturer strings from `lscpu`, `lshw`, or DMI, including Loongson, Phytium, Hygon, Zhaoxin, Huawei/HiSilicon, and Sunway; Huawei remains canonicalized to the project's existing `HiSilicon` CPU vendor bucket.
- `/proc/cpuinfo` fallback now covers LoongArch-style `cpu model` fields such as `Loongson-3A5000`.
- `/sys/devices/system/cpu/cpu*/topology` fallback now preserves Deepin-style CPU aggregate topology, including the `physical_package_id == -1` single-socket case seen on some non-x86 platforms, and combines it with `/sys/devices/system/cpu/online` to preserve Kylin-style online thread/core counts.
- `/sys/devices/system/cpu/cpu0/cache/index*` fallback now preserves Deepin-style sysfs cache fields and totals shared cache groups before exposing L1d/L1i/L2/L3/L4 cache values.
- `/sys/devices/system/cpu/*/cpufreq` fallback now preserves Kylin-style CPU max/min/current frequency values from sysfs when command sources lack frequency fields, uses `scaling_max_freq`/`scaling_min_freq` if `cpuinfo_max_freq`/`cpuinfo_min_freq` are unavailable, and current frequency uses the average of readable per-core `scaling_cur_freq` files when available.
- Deepin per-logical-CPU details and Kylin CPU governor/sensor controls were reviewed and left out of `CpuInfo`; aggregate L4 cache remains modeled because `lscpu`/sysfs expose it as hardware inventory data.
- Merged CPU names now strip Deepin-style trailing `x <count>` instance suffixes while intentionally preserving slash suffixes such as `Phytium D2000/8` that can be part of a domestic CPU model name.
- English/C locale is now enforced for real command execution, preventing localized command keys from silently breaking parsers such as `lscpu`.
- `/proc/hardware` fallback now recognizes `HUAWEI Kirin 990`, `kirin990`, and `HUAWEI Kirin 9006C`.
- `/sys/class/dmi/id` fallback now preserves BIOS/baseboard identity, including board version and asset tag, when `dmidecode -t 0,1,2,3` cannot run.
- BIOS probing now preserves firmware type (`uefi`/`bios`) and Secure Boot state from `/sys/firmware/efi` and efivars when available.
- BIOS probing now preserves BIOS Language Information fields from optional `dmidecode -t 13`, including language description format, installable languages, and currently installed language.
- `/proc/meminfo` fallback now preserves aggregate memory capacity when `dmidecode -t memory` cannot run.
- `lshw -class memory` now enriches successful DMI DIMM records by locator/serial and preserves DIMM slot, vendor, serial, part number, size, type, speed, width, product/description name, and source evidence; it remains the first DIMM fallback when `dmidecode -t memory` cannot run.
- EDAC sysfs memory fallback now preserves DIMM label/location, memory type, and size from `/sys/devices/system/edac/mc/mc*/dimm*` before falling back to Kylin FT1500a sysfs, device-tree, or aggregate `/proc/meminfo`.

- Raw SPD EEPROM fallback now reads `/sys/bus/i2c/drivers/eeprom/*/eeprom`, `/sys/bus/i2c/drivers/ee1004/*/eeprom`, and `/sys/bus/nvmem/devices/*/nvmem` when command SPD decoding is unavailable, preserving DDR4 DIMM size/common manufacturer IDs/type/speed/serial/part number and DDR5 identity fields (type, common module manufacturer, serial, part number) where the EEPROM bytes are parseable; the common manufacturer map now covers Samsung, SK Hynix, Micron, Crucial, Kingston, Ramaxel, ADATA, Longsys, Kimtigo, Nanya, Winbond, Powerchip, YMTC, CXMT, UNIC, UniIC, GigaDevice, Gloway, KingSpec, Netac, and Biwin and is exposed on memory device `vendor`.

- Empty successful `dmidecode -t memory` or `lshw -class memory` output now produces a `source_empty` warning and continues to lower-priority memory fallbacks.
- `/sys/class/power_supply` fallback now preserves battery identity, capacity, energy, voltage, temperature, cycle count, and present state when UPower cannot run.
- Battery probing now normalizes `LGC`/`LG Chem` vendor strings consistently across UPower and sysfs sources.
- Network probing now preserves IPv4/IPv6 addresses from `ip -j addr` when available; `/sys/class/net` preserves interface name, MAC, operstate, speed, duplex, wireless/ethernet type/capabilities, kernel driver, PCI bus identity, and kernel modules, including when `ip -j link` cannot run.

- `lshw -class network` enrichment now preserves network adapter product/model, vendor, PCI bus address, capacity-derived speed, driver version, and firmware when available without suppressing the base `ip`/sysfs path.
- Audio probing now preserves PCI bus address, vendor/device/subsystem/class IDs, and kernel module names from sound-card sysfs when available.

## Exception Handling Audit

Absorbed and preserved:

- `hw-source` classifies command/file errors as `Missing`, `PermissionDenied`, `Timeout`, or `Failed`, and runs commands with stable English/C locale environment.
- CPU treats `lscpu`, `lshw`, and `dmidecode` as optional and emits warnings for failed or empty-success sources while still producing a CPU when any useful source exists.
- Memory emits `source_empty` for successful but unparsable DMI/lshw/decode-dimms/raw-SPD memory sources, warns when raw SPD paths are unavailable, and continues through lower-priority SPD, EDAC, FT1500a sysfs, device-tree, and procfs fallbacks; DDR5 raw SPD size/speed deep decoding is intentionally outside local Deepin/Kylin parity scope.
- Monitor treats `xrandr --verbose` and sysfs EDID as optional, emits `source_empty` for empty successful `xrandr --query` parses, and continues after bad EDID with `edid_parse_failed` warnings.
- USB preserves the missing/failed `lsusb` warning while still emitting devices from `/sys/bus/usb/devices/*` when usable sysfs device directories exist; `lsusb -v` is optional enrichment and missing/failed verbose output does not suppress base USB devices.
- Storage treats `smartctl -a -j` as optional enrichment and preserves SMART data, including NVMe health counters, from parseable stdout even when smartctl exits non-zero.
- Audio preserves the missing/failed `/proc/asound/cards` warning while still emitting ALSA card devices from `/sys/class/sound/card*` when present, with sysfs driver/module/vendor/subsystem/PCI enrichment where available.
- Bluetooth preserves the missing/failed `hciconfig -a` warning while still emitting controllers from `/sys/class/bluetooth/hci*` when usable sysfs controller directories exist, and emits `source_empty` when successful `hciconfig` output parses no controllers.
- Input preserves the missing/failed `/proc/bus/input/devices` warning while still emitting basic event devices from `/sys/class/input/event*` or `hwinfo --keyboard`/`--mouse` when present, and emits `source_empty` plus fallback when proc input parsing yields no devices.
- Camera preserves the missing/failed `v4l2-ctl --list-devices` warning while still emitting devices from `/sys/class/video4linux/video*` when usable sysfs video nodes exist; optional format capability enrichment does not suppress base camera devices when unavailable.
- CD-ROM preserves the missing/failed `/proc/sys/dev/cdrom/info` warning while still emitting optical drive nodes and available identity fields from `/sys/class/block/sr*`, `lshw -class disk`, or `hwinfo --cdrom` when present, and emits `source_empty` plus fallback when proc cdrom parsing yields no drives.
- PCI preserves the missing/failed `lspci -nn -k` warning while still emitting PCI ID and driver devices from `/sys/bus/pci/devices/*` when present.
- GPU preserves the missing/failed `lspci -nn -k` warning and emits `source_empty` for successful empty lspci parses while still emitting GPU devices from display-class `/sys/bus/pci/devices/*` with available driver data when present; optional `lshw -class display` enrichment does not suppress the base lspci/sysfs path when missing.
- Printer preserves the missing/failed `lpstat -a` warning while still emitting queue/URI devices from `lpstat -v` when available, emits `source_empty` plus URI fallback when `lpstat -a` parsing yields no queues, and preserves default destination data from `lpstat -d` when available.
- Fake runners and fixture tests cover missing commands, permission-denied DMI, CPU `lscpu`/procfs disagreement, bad EDID, duplicate sysfs connector handling, ambiguous sysfs connectors, and numeric GPU vendor IDs.

Still weak:

- Printer and CD-ROM still have limited fallback/enrichment coverage compared with the reference projects; audio still has limited SoC vendor alias coverage; camera still lacks hwinfo enrichment.

## Deferred Items

These are not fully implemented yet and should remain tracked:

- P1: continue adding warning-on-empty-parse for additional parsers where command success does not mean usable data.
- P2: add network NM DBus enrichment, storage attribution for same-media multi-controller systems where sysfs cannot expose a readable PCI ancestor, broader SPD manufacturer-ID friendly-name coverage beyond the current common map, richer multi-interface USB modeling, camera hwinfo enrichment, broader audio SoC alias coverage, and Bluetooth hwinfo/DBus enrichments. DDR5 raw SPD size/speed decoding and richer SPD timing fields are not tracked here because local Deepin/Kylin do not implement that depth.
- P3: optional heavy display/GPU sources such as broader `hwinfo` and deeper renderer-to-PCI attribution for same-vendor GPUs whose renderer names do not uniquely identify a model.
